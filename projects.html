SDF-Molecule-Viewer

Project Link

Languages: Python, C, JavaScript, SQL, HTML, CSS
Technologies/Libraries: jQuery, SWIG, SQLite, GNU Make, Regex

This project is full-stack web application created as part of the "Software Systems Development and Integration" (CIS 2750) course at the University of Guelph.
The version of the project here is an upgraded version of the one I submitted for the course.
Here is what the project can do:

  • Allows the user to upload a .sdf file which will be parsed using Regex expressions
  • The parsed data will be added to an SQLite database, where it can be later used to generate an SVG display of a molecule at the users request
  • The user is able to customize the SVG by inputting information about the types of atoms in the molecule through an HTML form

MIX-Interpreter

Project Link

Languages: Go
Technologies/Libraries: GNU Make

This project is currently in-progress. It is an interpreter for the MIX assembly language described in Dr. Donald Knuth's "The Art of Computer Programming" Volume 1.
This work is also my first experience with the Go programming language.
It is not currently functional, but eventually I intend for it to be able to do the following things:

  • Perform syntax checking for all MIX instructions
  • Additional commands to the language will be added for debugging purposes, so that the user can see crucial information about the program during runtime
  • When the interpreter is run, there will be flag options such as allowing the user to skip syntax checking, as well as having the program print out the state of the RAM after it has executed

SSBM-Comparator

Project Link

Languages: Java
Technologies/Libraries: Gradle, Java Swing

This project was my first ever attempt at writing software outside of school (I started it shortly after finishing my first two semesters of computer science).
It was also my first attempt at learning a programming language by myself (Java).
While this project is not a good reflection of my current skillset, I do think it is worth putting on this website to show how much I have improved since.
Here is what the project is able to do:

  • Compare data of up to four Super Smash Bros. Melee characters at once
  • Compare the data of a character to the average of all characters
  • Rank (sort) all of the damaging moves in the game by their: -> Damage -> Startup Frames -> Active Frames -> Ending Frames
  • Rank (sort) character attributes (such as fast falling speed), as well as data about non-damaging moves (such as spot dodge)
